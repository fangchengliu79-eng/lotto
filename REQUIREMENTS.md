## 一、系统概述

多模型集成的彩票号码预测系统，支持**大乐透 (DLT)** 和 **双色球 (SSQ)** 两种彩种。通过 Streamlit Web 界面提供操作，顶部 tabs 切换彩种。

---

## 二、核心规则

### 2.1 大乐透 (DLT)
| 属性 | 值 |
|------|-----|
| 前区 | 01~35 选 **5** 个不重复 |
| 后区 | 01~12 选 **2** 个不重复 |
| 总组合 | C(35,5) × C(12,2) = 21,425,712 |
| 开奖 | 每周一、三、六 |
| CSV列 | period, front_1~front_5, back_1~back_2 |

### 2.2 双色球 (SSQ)
| 属性 | 值 |
|------|-----|
| 红球 | 01~33 选 **6** 个不重复 |
| 蓝球 | 01~16 选 **1** 个 |
| 总组合 | C(33,6) × C(16,1) = 17,721,088 |
| 开奖 | 每周二、四、日 |
| CSV列 | period, red_1~red_6, blue_1 |

### 2.3 投注方式（仅供参考，预测系统输出单式）
- **单式**：直接指定6个红球+1个蓝球（系统输​​出格式）
- **复式**：指定超过6个红球或超过1个蓝球
- **胆拖**：指定胆码+拖码

## 三、UI/UX 要求

### 3.1 主题风格（最终确定）

- **彩种切换**：通过 `st.tabs()` 在页面**顶部**实现"🎯 大乐透 / 🔴 双色球"标签页

- **子页面导航**：每个彩种内部用水平 radio 切换 4 个页面（预测结果 / 数据分析 / 预测历史 / 回测）

### 3.3 隐藏元素
- 隐藏 Streamlit 顶部默认蓝色工具栏 (`header[data-testid="stHeader"]`)
- 隐藏 Streamlit 默认的 "Deploy" 和管理按钮

### 3.4 号码展示
- 号码球 46px 圆形，monospace 字体
- 每组 5 个卡片横向排列
- 每个卡片包含：组号、号码球、得分、分析理由
- 命中号码用**黄色光圈**标记（不是绿色）

## 四、推荐生成算法要求

### 4.1 核心原则（用户多次强调）
1. **排名不变，推荐不变** — 如果算法评分排名没变化，点"重新计算"不应刷新号码
2. **概率分布驱动** — 使用历史数据的真实概率分布进行加权采样，不搞评分/融合/强制去重
3. **纯统计学方法** — 不要人为干预（不需要连号奖励、跨度惩罚、热区冷区切割）
4. **得分透明** — 得分 = 概率百分比直接相加，用户可以理解

### 4.2 算法演进史（供参考）
| 阶段 | 方法 | 问题 |
|------|------|------|
| v1 | 随机采样+过滤 | 号码全堆在小号 |
| v2 | 趋势追踪(热区60%+冷区25%) | 人为主观切割 |
| v3 | 融合评分+多样性惩罚 | 评分导致所有组都选33 |
| v4 (当前) | **纯概率分布加权采样** | 用户认可 |

### 4.3 最终算法
```python
# 使用 FrequencyModel 的真实概率分布
main_probs = freq_model.main_probs   # 35个号码的真实概率
sub_probs = freq_model.sub_probs     # 12/16个号码的真实概率

# 加权采样（不放回，一次抽一个）
for each group:
    for 5/6 times:   # DLT抽5次，SSQ抽6次
        pick a number with probability = main_probs[number]
        remove it and renormalize probabilities
    
    for 2/1 times:   # DLT抽2次，SSQ抽1次
        pick a number with probability = sub_probs[number]
        remove it and renormalize probabilities

# 得分 = 概率百分比的直接和
score = sum(main_probs[n] for n in group) + sum(sub_probs[n] for n in group)
score *= 100  # 转为百分比显示
```

### 4.4 历史数据要求
- **200条历史记录**（非30条）
- 双色球：30条真实zhcw数据 + 170条扩展（符合真实规律）
- 大乐透：原有数据 + 扩展至200条
- 扩展数据必须保持：连号率、区间分布、跨度等核心指标与真实开奖一致

### 4.5 数据获取
- 优先尝试 zhcw.com（使用浏览器工具提取Vue.js渲染后的数据）
- 退回到 500.com
- 再退回到本地 CSV

## 五、页面功能要求

### 5.1 预测结果页面
- 展示当前期预测的 5 组推荐号码（号码球形式）
- 已开奖的显示实际开奖号码+命中黄色高亮
- 显示各组得分
- **首次加载自动生成预测**（不显示"生成首期预测"按钮）
- 📅 **生成下一期预测** 按钮
- 🔄 **重新计算** 按钮（排名变了才更新，变了显示"已刷新"，没变显示"保持不变"）
- 显示比对统计（最佳命中、平均命中）

### 5.2 数据分析页面
- 号码频率表（前区/红球 + 后区/蓝球）
- 热号 TOP5 / 冷号 TOP3
- 和值均值、跨度均值、AC 值、连号率、常见奇偶比

### 5.3 预测历史页面
- 所有预测记录可展开
- 每组号码用**号码球展示**（非纯文本表格）
- 命中号码用黄色光圈标记
- 显示各组的命中统计

### 5.4 回测页面
- 滚动验证模型准确率
- 显示测试期数、平均命中、最优命中率
