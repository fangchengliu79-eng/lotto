## 三、数据流

```
1. app.py 启动 → data_fetcher.update_data()
   → 从500.com/CWL抓取HTML/JSON
   → 解析为DataFrame，按期号降序
   → 缓存到 ssq_history.csv / dlt_history.csv

2. → models/ensemble.py._get_model_probs(df)
   → 用所有历史期计算8个概率分布
   → 保存到 self._model_main_probs, self._model_sub_probs
   → 返回贝叶斯融合后的ensemble概率

3. → generate_recommendations()
   → 5种策略独立运行
   → 每种策略：构建融合概率 → 加权采样500次候选
   → 每次候选用CRF评分选最优
   → 返回5组推荐号码

4. → utils/predictions.save_prediction()
   → 保存到 predictions.json
   → 自动比对已有开奖数据

5. app.py 展示 → 每组卡片：策略标签 + 号码球 + 得分 + 说明
```

## 四、8个概率模型（_get_model_probs中计算）

每个模型为0~1之间的概率分布，sum=1。

| # | 模型名称 | 核心公式 | 含义 |
|---|---------|---------|------|
| 1 | **frequency** | (count+1)/(total+N) | 全历史频率（Laplace平滑） |
| 2 | **decay_frequency** | Σ(0.98^idx × occurrence) / Z | 衰减加权频率，近期权重大 |
| 3 | **sliding_window** | 同上，但只算最近50期 | 近期趋势快照 |
| 4 | **bayesian** | (count+1)/(total+N) | 同frequency（Beta(1,1)先验） |
| 5 | **poisson** | 1 - exp(-gap/λ) | 泊松逾期概率，越高越久未出 |
| 6 | **markov_chain** | P(n₁|n₀) 转移概率 | 基于历史转移模式 |
| 7 | **time_series** | α × occurrence + (1-α) × prev | 指数平滑，α=0.3 |
| 8 | **gap_recency** | sigmoid(z_score × 0.8) | 间隔Z-score，衡量缺席异常度 |

### 贝叶斯融合（代替线性平均）

```
线性平均:  P = Σ(w_i × P_i)            -- 矛盾证据会"和稀泥"
贝叶斯融合: P ∝ exp(Σ w_i × log(P_i))  -- 矛盾证据互相削弱
```

融合权重：

| 模型 | 权重 |
|------|------|
| decay_frequency | 0.20 |
| sliding_window | 0.18 |
| gap_recency | 0.12 |
| poisson | 0.15 |
| markov_chain | 0.15 |
| time_series | 0.12 |
| bayesian | 0.08 |

## 五、5种推理策略

每个策略独立运行，使用不同的融合概率，互不干扰。

### 策略1: 🔥 高频热号

| 项目 | 内容 |
|------|------|
| 融合模型 | decay_frequency(70%) + gap_recency(30%) |
| 推理逻辑 | 衰减加权后出现最多的号码（不是原始全频） |
| 采样 | 加权随机500次 |
| 评分 | CRF全局评分（结构优先） |
| 适用场景 | 找近期稳定的高频号码 |

### 策略2: 📈 近期趋势

| 项目 | 内容 |
|------|------|
| 融合模型 | sliding_window + trend_boost(涨幅) + decay_frequency |
| 推理逻辑 | 近期占比 - 历史占比 > 0 → 正在走热的号码 |
| 采样 | 加权随机500次 |
| 评分 | CRF全局评分 |
| 适用场景 | 抓最近开始升温的号码 |

### 策略3: ❄️ 追冷

| 项目 | 内容 |
|------|------|
| 范围 | 频率最低的40%号 = 冷号池 |
| 冷号评分 | gap_recency(70%) + recent_share(30%) |
| 非冷号 | 低权重(15%) |
| 推理逻辑 | 冷号中找"回补信号"最强的——不是单纯最久，而是Z-score异常度高且近期有升温 |
| 采样 | 加权随机500次 |
| 评分 | CRF全局评分 |
| 适用场景 | 找冷门号中最可能回暖的 |

### 策略4: 🔗 马尔可夫

| 项目 | 内容 |
|------|------|
| 基础 | markov_chain转移概率 |
| 增强 | 上期号码×1.2、上期邻域(±2)×1.15 |
| 融合 | markov(70%) + decay_frequency(30%) |
| 推理逻辑 | 上期开了什么号码 → 这些号及邻域的转移概率最高 |
| 采样 | 加权随机500次 |
| 评分 | CRF全局评分 |
| 适用场景 | 利用号码的聚集效应和延续性 |

### 策略5: ⚖️ 平衡策略

| 项目 | 内容 |
|------|------|
| 范围 | 排除最热30% + 最冷30%，选中段40% |
| 邻域 | 紧邻中段的前后10%给半权 |
| 评分 | CRF全局评分（结构质量权重最高） |
| 推理逻辑 | 不追热不追冷，纯结构合理 |
| 适用场景 | 当没有明显趋势时的安全选择 |

## 六、CRF全局组合评分（核心创新）

不是评估单个号码，而是评估**整个号码组合的结构质量**。

### 评分公式

```
final_score = base_prob 
            - consec_penalty    (>=3连号惩罚)
            - sum_penalty       (和值偏离均值惩罚)
            - odd_penalty       (奇偶不平衡惩罚)
            - span_penalty      (跨度偏离合理范围)
            - section_penalty   (区间扎堆惩罚)
```

### 惩罚项详解

| 惩罚项 | 公式 | 阈值 |
|-------|------|------|
| 连号 | max(0, max_streak-2) × 0.04 | 3+连号开始扣分 |
| 和值 | abs(actual-expected)/expected × 0.03 | 偏离越大扣越多 |
| 奇偶 | min(abs(odd-expected)/expected × 0.03, 0.03) | 上限0.03 |
| 跨度 | abs(span-expected_span)/expected_span × 0.02 | 偏离合理范围扣分 |
| 区间 | (max_section-min_section)/count × 0.02 | 号码扎堆在同一区间扣分 |

### 关键意义

**传统方法（sum of probs）**：
选出概率最高的6个号码 → 可能全是20+大号 → 结构失衡

**CRF打分**：
候选组合 [05 12 18 23 27 31] → 和值116/奇偶3:3/跨26 → 综合高分
候选组合 [20 21 22 23 24 25] → 和值135/奇偶3:3/跨5 → 跨度惩罚高分扣完

## 九、预测生命周期

```
创建预测 (active)
    │
    ├── 开奖数据尚未获取 → 保持 active
    │
    └── 新的开奖数据加入 → auto_compare_latest()
            │
            ├── 比对命中 → 标记 completed
            │             存储 actual_draw + matches + summary
            │
            └── 未命中 → 保持 active

历史记录: 最多保留10条 (active + completed)
新预测覆盖同一期号的旧预测
```

## 十一、当前已知局限 & 可改进方向

### 算法层面

| 局限 | 说明 | 潜在改进 |
|------|------|---------|
| 加权采样次数固定(500次) | 可能漏掉更好的组合 | 自适应采样：根据概率分布的熵决定采样次数 |
| CRF权重人工设定 | consec_penalty=0.04等是拍脑袋 | 从历史数据学习CRF权重（优化惩罚系数） |
| 5策略无动态权重 | 每种策略同等重要 | 根据近期命中率动态给策略加权 |
| 衰减系数固定0.98 | 所有号码同速衰减 | 对不同号码类型用不同衰减率 |
| 无位置编码 | 不考虑号码在序列中的位置模式 | 加入ball-position频率分析（个十位模式） |
| 无外部特征 | 只用历史开奖数据 | 加入日期/节气/星期等元数据 |

### 工程层面

| 局限 | 说明 |
|------|------|
| 无缓存热启动 | 每次启动都重新计算所有概率 |
| 无并发预测 | 预测和UI渲染串行 |
| 无AB测试框架 | 不能对比不同策略版本的效果 |
| 日志分散 | 没有统一的指标收集 |

### 数据层面

| 局限 | 说明 |
|------|------|
| 仅500期数据 | 更多历史数据可能提升低频信号可靠性 |
| 单一数据源 | 500.com/CWL可能宕机或改版 |

## 十二、与国际化方法的对比

| 方法 | 本系统 | 国际最佳实践 |
|------|--------|------------|
| 频率分析 | ✅ decay_frequency(0.98^期) | ✅ 同等级 |
| 滑动窗口 | ✅ 50期 | ✅ 同等级 |
| 贝叶斯融合 | ✅ log-space加权乘积 | ✅ 同等级 |
| CRF全局解码 | ✅ 组合评分 | ⚠️ 真CRF需要训练 |
| 间隔分析 | ✅ Z-score+sigmoid | ✅ 同等级 |
| 马尔可夫 | ✅ 1阶转移 | 🔼 可升级为聚类马尔可夫 |
| 深度学习 | ❌ 无 | LSTM/Transformer |
| 量子特征 | ❌ 无 | Qiskit |
| K-Means聚类 | ❌ 无 | 聚类调制 |
